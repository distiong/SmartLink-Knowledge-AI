# AI智能知识库问答系统（RAG\+知识图谱）前后端项目需求说明书

## 一、项目概述

本项目是一套基于SpringBoot3 \+ Vue3的**企业级AI智能问答演示系统**，采用第三方大模型API实现AI能力，无需本地部署大模型。系统集成**大模型通用对话、RAG知识库问答、向量语义检索、知识图谱关系查询、文档库管理**核心能力。

项目定位：技术演示型落地项目，完整还原当下企业最主流的AI落地方案——大模型API \+ 向量检索RAG \+ 知识图谱，可用于技术展示、面试项目、毕业设计、技术复盘。

## 二、整体技术栈（最新成熟企业版）

### 2\.1 后端技术栈

- 核心框架：SpringBoot 3\.2\.x、SpringAI 1\.0\.x

- JDK版本：JDK17（企业主流稳定版）

- AI能力：第三方大模型API（OpenAI/通义千问/文心一言任选）、云端Embedding向量化API

- 向量数据库：Milvus 2\.4（企业级向量索引、HNSW索引、语义检索）

- 知识图谱：Neo4j 5\.x（实体关系存储、图谱查询、关系推理）

- 业务数据库：MySQL 8\.0

- 工具依赖：Hutool、PDFBox、POI（文档解析）、Lombok

### 2\.2 前端技术栈

- 基础框架：Vue3 \+ Vite

- UI组件库：Element Plus

- 图表可视化：ECharts（知识图谱关系可视化）

- 网络请求：Axios

- 辅助工具：Vue Router、Pinia

## 三、系统整体架构

前端页面层 → SpringBoot后端服务层 → AI能力层 → 数据存储层

1\. 前端：提供可视化操作页面、对话界面、文档管理、图谱可视化界面

2\. 后端：统一接收前端请求、文档解析、分块、向量化、向量入库、RAG检索、大模型调用、图谱CRUD

3\. AI层：调用第三方大模型Chat API、Embedding向量API

4\. 存储层：MySQL（业务数据）、Milvus（向量索引\+知识库向量）、Neo4j（知识图谱实体关系）

## 四、核心功能需求（前后端全覆盖）

### 4\.1 通用AI对话模块

功能描述：纯大模型对话，无知识库关联，用于基础AI能力演示。

**前端需求**：

- 对话输入框、发送按钮

- 对话消息列表展示（区分用户提问、AI回答）

- 支持清空对话、实时加载loading状态

**后端需求**：

- 接收用户文本，调用第三方大模型Chat API

- 返回AI回答文本，统一返回格式

- 支持配置temperature、模型参数

### 4\.2 RAG知识库问答模块（核心核心）

功能描述：企业知识库落地核心，实现文档入库、向量化、向量索引、语义检索、溯源问答。

**完整业务流程**：用户上传文档 → 后端解析文本 → 文本分块Chunk → 调用Embedding生成向量 → Milvus建立HNSW索引存储 → 用户提问 → 问题向量化 → 相似度检索召回上下文 → 拼接提示词 → 大模型生成带溯源答案

**前端需求**：

- 文档上传组件：支持TXT、PDF、Word格式上传

- 知识库文件列表：展示已上传文档名称、上传时间

- RAG专属问答界面

- 问答结果展示：问题、AI答案、引用的原文片段（溯源展示）

**后端需求**：

- 文档解析：PDF/Word/TXT文本提取

- 文本分片策略：固定长度分片、保留上下文

- 调用第三方Embedding接口生成向量

- Milvus向量库新增、建立索引、语义相似度检索

- RAG提示词工程：约束大模型仅基于知识库回答，禁止幻觉

- 返回答案\+原文上下文溯源内容

### 4\.3 向量检索演示模块

功能描述：独立演示向量语义检索能力，区别于传统关键词搜索。

**前端需求**：输入文本，展示TopN相似文档片段、相似度分值

**后端需求**：基于Milvus向量索引，余弦相似度检索，返回匹配片段与相似度

### 4\.4 知识图谱模块

功能描述：实现实体、关系的创建与可视化查询，演示知识推理能力。

**前端需求**：

- 图谱数据录入表单：主体、关系、客体

- 图谱查询输入框

- ECharts可视化图谱节点与关系连线

- 关系结果列表展示

**后端需求**：

- 接收前端实体关系数据，执行Cypher语句创建节点与关系

- 根据实体名称查询关联的所有关系、关联节点

- 封装图谱可视化所需节点、关系数据结构

### 4\.5 文档管理模块

**前端**：文档列表、删除文档、查看文档详情

**后端**：文档信息入库、删除向量库对应数据、删除业务数据

## 五、前后端接口规范（统一约束）

所有接口统一返回格式：

```Plain Text
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

### 后端核心接口清单

1. **通用AI对话**：GET /api/ai/chat

2. **知识库文档上传**：POST /api/rag/upload

3. **RAG智能问答**：GET /api/rag/chat

4. **向量语义检索**：GET /api/rag/search

5. **知识图谱创建关系**：POST /api/graph/create

6. **知识图谱查询**：GET /api/graph/query

7. **文档列表查询**：GET /api/document/list

8. **文档删除**：DELETE /api/document/delete

## 六、数据库设计

### 6\.1 MySQL 文档表（ai\_document）

- id：主键

- file\_name：文档名称

- file\_type：文档类型

- content：文档纯文本内容

- create\_time：上传时间

### 6\.2 Milvus 向量集合

存储字段：文档片段文本、向量值、文档ID、分片序号，索引类型：HNSW

### 6\.3 Neo4j 图谱结构

节点：自定义实体节点（含name属性），关系：自定义关系名称
